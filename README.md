## Overview

Dense wildfire smoke obscures satellite imagery critical for evacuation coordination and damage assessment. This project explores two deep learning approaches to restore smoke-degraded Sentinel-2 imagery from British Columbia wildfire events, grounded in the physics of the Atmospheric Scattering Model (ASM).

**Key Results:**

| Model | PSNR | SSIM | Parameters |
|-------|------|------|------------|
| DGTV | 27.52 dB | 0.8282 | ~120K |
| AOD-Net | 27.03 dB | **0.9255** | ~10K |

## Scripts & Modules

### `Preprocessing_Augmentation-final-code.ipynb`
Loads raw Sentinel-2 `.SAFE` files, applies gamma correction, extracts 256×256 patches, synthesizes realistic smoke via ASM (Brown/White/Grey profiles), filters low-quality patches (mean intensity < 0.2), and applies 8× augmentation to produce ~9,600 clean/smoky pairs saved as `.npy` arrays.

### `DGTV.ipynb`
Implements Deep Graph Total Variation — a hybrid of a lightweight 4-layer feature CNN and an analytical graph total variation filter. Includes 3-stage ablation experiments (feature dim K → optimizer → GTV layers T), training loop, and inference on real smoky Sentinel-2 imagery.

### `AOD_NET.ipynb`
Implements AOD-Net (Li et al., ICCV 2017) adapted for wildfire smoke. A 5-layer CNN with multi-scale concatenation directly encodes the ASM physics: `x̂ = K(y)·y − K(y) + 1`. Includes training loop and inference on real smoky imagery.

## Preprocessing Pipeline

> All preprocessing is implemented in `Preprocessing_Augmentation-final-code.ipynb`.

### Step 1 — Load `.SAFE` Files & Extract RGB Bands

Sentinel-2 `.SAFE` packages store each spectral band as a separate `.jp2` file inside a fixed directory layout. Only the three 10m visible bands are used:

| Band | Channel | Resolution |
|------|---------|------------|
| B02 | Blue | 10m |
| B03 | Green | 10m |
| B04 | Red | 10m |

Each scene is loaded with `rasterio` into a `(H, W, 3)` float32 array, then **center-cropped to 1024×1024** to remove tile boundary artefacts and fix the spatial footprint across all scenes.

```
File path: GRANULE/*/IMG_DATA/R10m/*{band}*.jp2
```

---

### Step 2 — Normalization (Gamma Correction)

Normalization is applied to the **full 1024×1024 scene before patch extraction** so that all patches derived from the same image share a consistent intensity scale.

```
normalized = (img - min) / (max - min + 1e-8)   # min-max rescale to [0, 1]
output     = normalized ^ (1 / 2.2)              # gamma correction (sRGB standard)
```

- **Min-max rescaling** maps raw DN values (0–4095) to [0, 1] using the global scene extremes. A small epsilon (1e-8) prevents division by zero on flat regions.
- **Gamma 2.2** (sRGB standard) brightens mid-tones to match display encoding, making synthesized smoke visually realistic.
- After gamma correction the minimum pixel value is ~0.146, which informs the dark-patch filter threshold in Step 3.

---

### Step 3 — Patch Extraction (256×256)

**150 random 256×256 patches** are sampled per image using `sklearn.feature_extraction.image.extract_patches_2d`. A fixed random seed (42 + image index) ensures reproducibility across runs.

Patches are converted from `(N, H, W, C)` (sklearn) to `(N, C, H, W)` (PyTorch convention).

Two dataset versions are produced in parallel:

| Version | Dark Patch Filter | Patches kept per image | Notes |
|---------|-------------------|----------------------|-------|
| **A** | None | 150 | Baseline — includes water bodies and shadow patches |
| **B** ✅ | mean < 0.2 → discard | ~112 (~25% removed) | Removes information-poor patches |

**Why threshold at 0.2?**  
At 0.1 nothing gets filtered (post-gamma minimum is 0.146). At 0.2, ~400 patches (25%) are removed — water bodies, deep shadows, and cloud-shadow areas where the ASM haze model does not produce realistic smoke and would mislead training.

---

### Step 4 — Smoke Synthesis (Physics-based ASM)

Synthetic smoke is applied patch-by-patch using the **Koschmieder Atmospheric Scattering Model**:

```
I_hazy = I_clean × (1 - t) + A × t
```

| Symbol | Meaning |
|--------|---------|
| `I_hazy` | Output smoky patch |
| `I_clean` | Input clean patch |
| `t` | Transmission coefficient — sampled from Uniform(0.2, 0.7) independently per patch |
| `A` | Atmospheric light vector (RGB) — randomly chosen per patch from 3 profiles |

**Transmission range:**

| t value | Smoke level |
|---------|-------------|
| 0.2 | Light haze |
| 0.7 | Dense smoke (matches 20230830 Sentinel-2 acquisition) |

**Three smoke color profiles** derived from real BC wildfire Sentinel-2 imagery:

| Profile | R | G | B |
|---------|---|---|---|
| Brownish smoke | 0.80 | 0.75 | 0.65 |
| White smoke | 0.90 | 0.90 | 0.90 |
| Grey smoke | 0.85 | 0.82 | 0.78 |

Each patch independently samples both `t` and the smoke profile, producing a diverse distribution of densities and tints. Output is clipped to [0, 1].

---

### Step 5 — Data Augmentation (×8, Training Only)

Each (noisy, clean) patch pair is expanded into 8 variants by applying the **same geometric transform to both patches simultaneously**, preserving ground-truth alignment.

| # | Transform |
|---|-----------|
| 1 | Original |
| 2 | Rotate 90° |
| 3 | Rotate 180° |
| 4 | Rotate 270° |
| 5 | Horizontal flip (left ↔ right) |
| 6 | Vertical flip (top ↔ bottom) |
| 7 | Horizontal flip + Rotate 90° |
| 8 | Vertical flip + Rotate 90° |

The augmented dataset is randomly shuffled to decorrelate augmented variants of the same patch across mini-batches.

---

### Step 6 — Save as `.npy`

| File | Shape | Description |
|------|-------|-------------|
| `train_clean_aug.npy` | (N×8, 3, 256, 256) | Augmented clean patches |
| `train_noisy_aug.npy` | (N×8, 3, 256, 256) | Augmented hazy patches |
| `test_clean.npy` | (20, 3, 256, 256) | Held-out clean patches |
| `test_noisy.npy` | (20, 3, 256, 256) | Held-out hazy patches |

**Train/test split:** Images 0–3 (4 scenes) → training, Image 4 (1 scene) → test (held-out scene).  
**Final dataset size:** ~1,200 patches × 4 images × 8 augmentations = **~9,600 pairs**

## Methods

### Atmospheric Scattering Model (ASM)

The physical basis for both smoke synthesis and AOD-Net's design:

```
I(x) = J(x) · t(x) + A · (1 − t(x))
```

| Variable | Description |
|----------|-------------|
| `I(x)` | Observed smoky image (input) |
| `J(x)` | Clean scene radiance (target) |
| `t(x)` | Transmission map (smoke density) |
| `A` | Global atmospheric light |

### DGTV — Deep Graph Total Variation
Combines a lightweight CNN (feature extraction) with an analytical graph total variation filter derived from convex optimization. Achieves competitive PSNR with 80% fewer parameters than DnCNN. Key limitation: the optimization objective assumes AWGN, which is physically mismatched to ASM-structured smoke.

### AOD-Net — All-in-One Dehazing Network
Directly encodes ASM into its architecture. A single CNN estimates the joint parameter `K(y)` and reconstructs the clean image as `x̂ = K(y)·y − K(y) + 1`, with no separate estimation of `t(x)` and `A`. Only ~10K parameters; achieves higher SSIM than DGTV.
